### 3.1. 기원과 영향력
<br/>
#### 3.1.3. 사업적 동인
<br/>
##### 3.1.3.1. 용량 계획
<br/>
용량 계획이란 조직의 IT 자원, 상품, 서비스의 미래 수요를 가늠하고 충족시키는 공정을 만한다. 이런 맥락에서 용량은 IT 자원이 주어진 시간 내에 달성할 수 있는 일의 최대치를 나타낸다. IT 자원의 용량과 수요 사이의 간극은 시스템을 비효율적(과잉 공급)이게 하거나 사용자의 요구사항을 충족시킬 수 없게(공급 부족) 만든다. 따라서 용량 계획은 예측 가능한 효율과 성능을 달성하기 위해 이 간극을 최소화하는 데 중점을 둔다. 용량 계획을 위한 전략은 다음과 같다.  

+ 선두 전략  
수요의 예측 범위 내에서 IT 자원 용량 추가

+ 지연 전략  
IT 자원이 최대 용량 사용에 도달했을 때 IT 자원 용량 추가

+ 사상 전략  
수요가 증가하는 만큼 조금씩 IT 자원 용량 추가  

용량 계획은 사용 부하의 변동을 예측해야 하기에 어려움이 있다. 인프라의 불필요한 사용없이 최대 사용량 요구에 균형을 맞춰야 한다. 예를 들어 최대 사용 부하를 충족시키기 위해 IT 인프라를 갖추는 것은 부적절하게 많은 재정 투자를 초래한다. 그러나 이때 투자를 조정한다면 용량 임계치가 낮아져 사용상 제약이나 트랜잭션 손실을 가져올 수 있다.  

##### 3.1.3.2. 비용 절감
<br/>
IT 비용과 비즈니스 성과를 직접적으로 비교하기는 어렵다. IT 환경의 성장은 최대 사용 요구량 평가에 맞춰 이루어진다. 새롭게 확장되는 비즈니스 자동화를 지원하는 데는 더 많은 투자가 필요하다. 자동화 솔루션의 잠재적 사용 능력은 인프라의 처리 능력에 제한되기 때문에 투자의 대부분은 인프라 확장에 쓰인다.  

여기서 새로운 인프라를 구축하기 위한 비용과 지속적으로 유지하는 비용, 이 두 가지가 설명되어야 한다. 운영 오버헤드는 IT 예산의 상당한 비중을 차지하며, 종종 선행 투자 비용을 초과한다.  

인프라와 관련한 운영 오버헤드는 다음과 같은 형태로 나타난다.  

+ 환경적 운영을 위한 기술 인력
+ 추가적인 테스트와 배포 사이클에 필요한 업그레이드와 패치
+ 전원 및 냉각 장치에 필요한 공공 요금과 자본금
+ 인프라 자원을 보호하기 위해 유지, 강화되어야 하는 보안과 접근 제한
+ 라이선스 관리와 지원 협약을 위한 관리자와 재장 담당자  

내부적인 기술 인프라를 유지하는 것은 기업의 예산에 복합적인 영향을 주는 책임을 수반한다. 결과적으로 IT 부서는 민감성과 수익성, 전체적 진화를 잠재적으로 저하시키면서 사업적 자원을 고갈시키는 요인이 된다.  

#### 3.1.4. 기술 혁신
<br/>
##### 3.1.4.1. 클러스터링
<br/>
클러스터는 한 시스템 내에 연결되어 있는 독립적인 IT 자원의 그룹이다. 클러스터의 중복과 시스템 대체 작동의 특성으로 인해 가용성과 신뢰성이 증가하고 시스템 실패율은 낮아진다.  

하드웨어를 클러스터링하기 위한 일반적인 전제 조건은 각 컴포넌트 시스템이 동일한 기종의 하드웨어와 운영체제로 이루어져야 한다는 것이다. 이는 한 컴포넌트가 실패했을 때 이를 비슷한 성능의 수준으로 다른 컴포넌트가 실패했을 때 이를 비슷한 성능의 수준으로 다른 컴포넌트가 대체할 수 있어야 하기 때문이다. 클러스터를 구성하는 컴포넌트 장비는 전용의 고속 통신 링크를 통해 동기화되어야 한다.  

내장된 중보과 대체 작동의 기본 개념은 클라우드 플랫폼의 핵심 개념이다.  

##### 3.1.4.2. 그리드 컴퓨팅
<br/>
컴퓨터 그리드는 컴퓨팅 자원이 하나 이상의 논리적인 풀로 구성된 플랫폼을 말한다. 이 풀은 서로 협동하여 고성능의 분산 그리드를 제공하는데, 이를 '슈퍼 가상 컴퓨터'라고 부르기도 한다. 그리드 컴퓨팅은 훨씬 느슨하게 결합되어 있고 분산되어 있다는 점에서 클러스터링과는 차이가 있다. 결과적으로 그리드 컴퓨팅 시스템은 서로 다른 기종이 지역적으로 분산되어 있는 시스템으로 클러스터 컴퓨팅 기반의 시스템으로 지원 불가능한 컴퓨팅 자원에 사용될 수 있다.  

그리드 컴퓨팅은 컴퓨팅 자원에 배포되는 미들웨어 계층에 기반을 둔다. 이러한 IT 자원은 워크로드 분배와 협동 기능을 구현하는 그리드 풀의 구성요소가 된다. 이 중간 계층은 부하 분산 로직과 대체 작동 제어, 자동 설정 관리 등을 포함하는데, 이는 클라우드 컴퓨팅 기술과 비슷하다(일부는 더욱 정교화되었다).  

##### 3.1.4.3. 가상화
<br/>
가상화는 IT 자원의 가상 객체를 만들어 사용하는 기술 플랫폼을 말한다. 가상화 소프트웨어 계층은 물리적인 IT 자원의 여러 개의 가상 이미지를 제공하여 프로세싱 용량을 여러 사람이 공유할 수 있게 한다.  

가상화 기술의 출현이 있기 전 소프트웨어는 정적인 하드웨어 환경에 고정되었다. 가상화로 구성된 환경에서 구동되는 에뮬레이션 소프트웨어로 하드웨어 요구사항을 시뮬레이션할 수 있어 가상화 프로세스는 소프트웨어와 하드웨어의 의존 관계를 해소한다.  

##### 3.1.4.5. 기술 혁신 vs. 기술을 가능하게 하는 것
<br/>
현재의 클라우드 기반 플랫폼에 기여하고 있는 여러 기술 영역들을 강조하는 것은 매우 중요한 일이다.  

+ 광대역 네트워크와 인터넷 아키텍처
+ 데이터 센터 기술
+ (현대) 가상화 기술
+ 웹 기술
+ 서비스 기술

### 3.2. 기본 개념과 용어
<br/>
#### 3.2.1. 클라우드
<br/>
클라우드는 원경으로 확장성 있고 측정된 IT 자원을 제공하기 위한 목적으로 고안된 별개의 IT 환경을 일컫는다. 클라우드 용어는 분산된 IT 자원에 원격으로 접속하는 네트워크인 인터넷의 은유적인 표현에서 발생했다. 클라우드 컴퓨팅이 IT 업계에 공식적인 영역이 되기에 앞서 클라우드(구름 모양의) 기호는 웹 기반 아키텍처의 명세나 문서에서 인터넷을 나타내는 데 사용했다. 이제는 클라우드 환경의 경계를 나타내는 데 사용된다.  

인터넷에서 '클라우드' 용어와 클라우드 기호를 구별하는 것은 중요하다. 원격 제공되는 IT 자원에 사용되는 특정한 환경으로서 클라우드는 명확한 경계를 갖는다. 인터넷을 통해 접근될 수 있는 개개의 클라우드가 존재한다.  

인터넷은 여러 웹 기반 IT 자원에 대한 개방된 접근을 허용하는 반면에 클라우드는 대개 특정 개인에게 속하며 제한된 접근을 허용한다.  

인터넷은 월드 와이드 웹(World Wide Web)을 통해 발생한 내용 기반의 IT 자원에 접근하게 한다. 반면 클라우드 환경에서 제공되는 IT 자원은 백 엔드에서의 처리 능력과 이러한 처리 능력에 대한 사용자 기반의 접속을 제공한다. 또 하나의 차이점은 클라우드는 인터넷 프로토콜과 기술 기반으로 이루어지지만 꼭 웹 기반일 필요는 없다는 것이다. 프로토콜은 이미 정의되고 구조화된 방식으로 컴퓨터가 통신하기 위한 기준과 방법을 말한다. 클라우드는 IT 자원을 원격으로 접속할 수 있게 하는 방식의 프로토콜을 모두 사용할 수 있다.  

흔하지는 않지만 악성 중개자 공격은 악성 클라우드 서비스 소비자 프로그램에 의해 수행될 수도 있다.  

#### 3.2.3. 온 프레미스
<br/>
클라우드는 원격으로 접근되는 개별적인 환경으로서 IT 자원의 배포를 위한 하나의 옵션에 해당한다. 전통적으로 IT 기업에서 제공되는 IT 자원은 조직의 경계 안에 있는 IT 엔터프라이즈 자체에 위치하는 것으로 간주되어왔다. '온 프레미스'는 "제어되는 IT 환경이 클라우드 기반이 아닌 것을 전제로 한다"는 것을 말한다. '온 프레미스'라는 용어는 IT 자원이 클라우드 기반의 대안으로 사용된다. 온 프레미스되는 IT 자원은 클라우드 기반이 될 수 없으며 반대 상황도 마찬가지다.  

주의 사항은 다음과 같다.  

+ 온 프레미스된 IT 자원은 클라우드 기반의 IT 자원에 접근하고 통신할 수 있다.
+ 온 프레미스된 IT 자원은 클라우드로 이관될 수 있으며, 이는 클라우드 기반의 IT 자원으로의 변화를 의미한다.
+ IT 자우너은 온 프레미스 환경과 클라우드 환경에 중복 배포될 수 있다.  

#### 3.2.4. 확장
<br/>
+ 수평적 확장 : 같은 형태의 IT 자원의 할당이나 배포
+ 외부 확장 : 자원의 수평적 할당
+ 내부 확장 : 자원의 수평적 배포
+ 수직적 확장 : IT 자원이 더 높거나 낮은 사양의 다른 자원으로 교체하는 것으로 교체가 일어날 때는 다운 타임이 발생하기 때문에 클라우드 환경에서는 일반적이지 않다.
+ 상향 확장 : 더 높은 사양의 IT 자원으로 교체하는 것
+ 하향 확장 : 더 낮은 사양의 IT 자원으로 교체하는 것

#### 3.2.5. 클라우드 서비스
<br/>
클라우드가 원격에서 접근 가능한 환경이긴 하지만 클라우든 내의 모든 IT 자원에 원격 접근이 가능한 것은 아니다. 예를 들어 클라우드 내의 데이터베이스나 물리적 서버는 같은 클라우드 안에 있는 IT 자원만 접근할 수 있다. 원격의 사용자가 접근할 수 있게 하려면 공개된 API를 이용한 소프트웨어 프로그램이 특별히 배포될 것이다.  

클라우드 서비스는 클라우드를 통해 원격으로 IT 자원에 접근 가능하도록 한다. 서비스 기반 아키텍처와 같이 서비스 기술 영역 하에 있는 다른 IT 영역과 달리, 클라우드 컴퓨팅의 문맥에서의 '서비스'라는 용어의 범주는 특히 방대하다. 클라우드 서비스는 메시징 프로토콜을 이용해 호출되는 기술적 인터페이스를 갖는 단순한 웹 기반 소프트웨어 프로그램으로 존재할 수도 있고, 관리자 도구나 더 큰 환경 및 IT 자원으로의 원격 접속 포인트일 수도 있다.  

클라우드 컴퓨팅의 동기는 원격에서 사용할 수 있는 기능을 제공하면서 다른 IT 자원을 캡슐화하여 서비스로서 IT 자원을 제공하는 데 있다. 클라우드 서비스의 여러 모델이 출현했는데, 대부분은 '서비스로서(as-a-service)'라는 접미사가 붙는다.  

클라우드 서비스 사용 조건은 서비스 수준 합의서(SLA)에 표현된다.  

SLA는 클라우드 기반 서비스의 서비스 품질(QoS, Quality-of-Service) 특성이나 행동양식, 제약사항, 프로비저닝 등을 명시한 클라우드 제공자와 클라우드 소비자간 서비스 계약서다.  

SLA는 가동시간, 보안 특성, 가용성, 신뢰성, 성능을 포함한 특정 서비스 품질 특성과 같은 IT 성과와 관련한 여러 측정 가능한 특성의 상세내역을 제공한다. 서비스의 구현은 클라우드 소비자에게 숨겨진 부분이기에 SLA는 중요한 명세서가 된다.  

#### 3.2.6. 클라우드 서비스 소비자
<br/>
클라우드 서비스 소비자는 클라우드 서비스에 접근할 때 소프트웨어 프로그램에 의해 부여되는 일시적인 런타임 역할이다.  

클라우드 서비스 소비자의 일반적인 형태는 서비스 계약이 맺어진 클라우드 서비스에 원격에서 접근할 수 있는 소프트웨어 프로그램과 서비스를 포함한다. 또한 소프트웨어가 구동되는 워크스테이션과 랩탑 컴퓨터, 모바일기기 등을 포함한다.  

### 3.3 목표와 이점
<br/>
#### 3.3.1. 투자 절감과 비례 비용
<br/>
낮은 가격으로 대량의 물건을 구입하는 도매업자와 비슷하게 공용 클라우드 제공자는 클라우드 소비자에게 매력적인 가격에 임대 패키지를 제공하여 IT 자원을 대량 획득하게 하는 비즈니스 모델에 기반을 둔다. 이는 조직이 인프라를 직접 구매하지 않고 성능 좋은 인프라에 접근할 수 있는 포문을 열었다.  

일반적으로 클라우드 기반 IT 자원에 투자하는 경제적인 목적은 하드웨어와 소프트웨어의 구입 및 유지 비용인 선행 IT 투자 비용을 절감하거나 제거하는 데 있다. 클라우드의 측정된 사용 특성이 예측했던 자본 지출을 대체하는 측정된 운영 지출(비즈니스 성과와 직접적으로 연관된)을 대변한다. 이는 비례 비용이라고 한다.  

선행된 재정 투자 계획을 제거하거나 축소하는 것은 기업을 축소시키며 핑요한 때마다 IT 자원을 할당하게 한다. 게다가 선행 자본 사용의 절감은 자본이 핵심 비즈니스 투자로 돌아가게 한다. 비용을 낮추는 기회는 주요 클라우드 제공자가 대형 데이터 센터를 배포하고 운용할 때 얻을 수 있다. 이런 데이터 센터는 부동산, IT 전문가, 네트워크 대역폭을 낮은 가격에 얻을 수 있는 곳에 위치하게 되며 이는 자본과 운영 비용의 절감을 가져온다.  

이와 같은 이유는 운영체제나 미들웨어 또는 플랫폼 소프트웨어, 애플리케이션 소프트웨어에도 적용된다. 여러 클라우드 소비자들이 IT 자원의 집합을 이용하고 공유하게 되어 가용성이 극대화된다. 운영 비용과 비효율성은 실용적 기술과 클라우드 아키텍처, 관리, 통제의 최적화 패턴을 적용하여 감소시킬 수 있다.  

클라우드 소비자가 얻는 이득은 다음과 같다.  

+ 단기간(예: 시간당 프로세서)을 기준으로 컴퓨팅 자원 사용료를 먼저 지급하고 더 이상 필요없을 때 컴퓨팅 자원을 방출하는 온디맨드식의 접근
+ 사용 가능한 컴퓨팅 자원에 제약이 없다는 인식, 이로 인해 프로비저닝을 준비할 필요가 없다.
+ 기가바이트 단위로 저장 디스크 공간을 확보하는 식의 미세한 IT 자원의 조절이 가능하다.
+ 인프라의 추상화로 애플리케이션을 장치나 장소에 구애받지 않고 필요할 때 쉽게 제거할 수 있다.  

클라우드 컴퓨팅의 경제적 이득을 따져보는 것은 쉽지만 실제 경제적으로 계산하고 측정하는 것은 복잡하다. 클라우드 컴퓨팅 도입 전략을 진행할지를 결정하는 것은 임대 비용과 구매 비용의 단순한 비교 이상의 문제다.  

클라우드가 제공하는 비용 절감의 한 축이 바로 IT 자원 공급의 기술적, 운영적 구현 상세내역이 클라우드 소비자에게 감춰지고 사용 대기(ready-to-use)나 규격품(off-the-shelf)으로 패키지화하여 제공하는 '서비스로서' 사용 모델이다. 제품에 기반한 이런 시비스는 온 프레미스되는 솔루션과 동등한 성능하에 비교했을 때 IT 자원의 개발, 배포, 관리를 간소화하고 신속하게 처리되게 한다. 때에 맞는 절약과 요구되는 IT 전문지식은 매우 중요한 요건으로 클라우드 컴퓨팅 도입의 정당성에 기여한다.  

#### 3.3.2. 확장성 증대
<br/>
클라우드는 집합적으로 사용될 때 더욱 영향을 미치도록 고안된 도구 및 기술과 더불어 IT 자원의 풀을 제공하여 클라우드 소비자의 요구가 있을 때나 직접적인 설정에 따라 클라우드 소비자에게 즉각적이고 동적으로 IT 자원을 배분한다. 이로서 클라우드 소비자는 클라우드 기반의 IT 자원을 처리 변동량과 피크에 맞춰 자동이나 수동으로 조절하는 권한을 갖는다. 처리할 요구사항이 감소하면 클라우드 기반의 IT 자원은 (자동 혹은 수동으로) 할당이 해제된다.  

IT 자원의 확장을 유연하게 하는 클라우드에 내재, 내장된 특성은 앞엇 언급한 비례 비용 이득과 직접적 연관이 있다. 확장과 관련해서는 자동화된 축소로 경제적 이득을 얻을 수 있을 뿐 아니라 IT 자원의 능력이 예측 불가한 사용량 요구를 충족시켜 사용량 임계치에 도달했을 때 발생할 수 있는 잠재적 손실을 피할 수 있게 한다.  

확장성 증대가 가져오는 이득은 사업적 동인 부분에서 설명한 용량 계획 전략과 관련이 있다.  

지연, 사상 전략은 클라우드가 IT 자원을 필요할 때마다 확장할 수 있기 때문에 가능한 것이다.  

#### 3.3.3. 가용성과 신뢰성 증대
<br/>
IT 자원의 가용성과 신뢰성은 눈에 보이는 비즈니스 이익과 직접적인 관련이 있다. 정전은 IT 자원이 사용자에게 개방되는 시간을 제한하므로 IT 자원의 사용과 수익을 제한한다. 바로 수정되지 않는 런타임 실패는 사용량이 많은 시기에 더 큰 영향을 받는다. 사용자의 요구에 맞춰 IT 자원을 공급할 수 없을 뿐 아니라 예측하지 못한 실패로 사용로부터의 신뢰성이 경감될 수 있다.  

전형적인 클라우드 환경의 특징은 정전 현상을 단축하거나 아예 제거할 수 있게 클라우드 기반 IT 자원의 가용성을 증대시키고 런타임 실패에 영향을 미치는 요소들을 최소화하여 신뢰성을 향상시켜서 광범위한 자원을 제공하는 고유한 능력에 있다.  

+ 가용성이 증대된 IT 자원은 더 오랜 시간 동안 접근 가능하다(예: 24시간 하루 중 22시간). 클라우드 제공자는 대개 높은 가용성을 보장하는 '탄력적인' IT 자원을 제공한다.  
+ 신뢰성이 증대된 IT 자원은 예외 발생 상황을 더욱 잘 방지하고 복구할 수 있다. 클라우드 환경의 모듈화된 아키텍처는 확장된 대체 작동을 지원하여 신뢰성을 증대시킨다. 조직이 클라우드 기반 서비스와 IT 자원 임대를 고려할 때 클라우드 제공자가 제공하는 SLA 주의 깊게 조사하는 것은 중요하다. 클라우드 환경이 매우 높은 가용성과 신뢰성을 제공하더라도 실제 계약된 의무사항을 기술한 SLA에 있는 보증 내역을 다 준수하지 못할 수 있다.  

+ 클라우드 환경은 실제 사용한 만큼만 청구되는 사용량당 과금 방식으로 임대되는 IT 자원의 풀을 제공하는 광범위한 인프라로 구성된다. 온 프레미스 환경과 비교할 때, 클라우드는 측정된 사용량에 비례하는 초기 투자 비용과 운영 비용을 절감할 수 있다.  
+ IT 자원을 확장하는 클라우드의 내재된 능력은 조직이 사용자의 요구를 거절하는 기 설정된 임계치에 제약을 받지 않고 예측 불가능한 사용량 변동에 맞춰 조절할 수 있게 한다. 반대로 요구되는 확장을 줄일 수 있는 클라우드의 능력은 비례 비용 이득과 직접적으로 관련이 있다.
+ IT 자원의 가용성과 신뢰성을 높일 수 있도록 클라우드 환경을 활용해서, 조직은 서비스 품질 보증을 증대시키고, 예측하지 못한 런타임 실패로 인한 비즈니스 손실을 경감하거나 제거할 수 있다.  

#### 3.4. 위험과 고려사항
<br/>
##### 3.4.1. 보안 취약성 증가
<br/>
비즈니스 데이터가 클라우드로 옮겨간다는 것은 데이터 보안에 대한 책임이 클라우드 소바자에게도 전가된다는 것을 의미한다. IT 자원을 원격으로 사용하는 것은 외부의 클라우드를 포함하려는 클라우드 소비자에게 신뢰 경계의 확장을 필요로 한다. 클라우드 소비자와 클라우드 제공자가 동일하거나 호환성 있는 보안 프레임워크를 자원하지 않는 이상(이는 공용 클라우드와는 다르다) 취약성을 끌어들이지 않고 신뢰 경계선을 확장시키는 보안 아키텍처를 구축하는 것은 매우 어려운 일이다.  

신뢰 경계가 중복되는 것은 클라우드 제공자가 클라우드 소비자의 데이터에 접근하는 특권을 갖는 것과 관련이 있다. 어떤 데이터가 안전하지에 대한 척도는 클라우드 소비자와 제공자에게 동시에 적요되는 보안 제어 및 정책에 종속된다. 나아가, 클라우드 기반의 IT 자원은 대개 공유되기 때문에 다른 클라우드 소비자들로부터 겹치는 신뢰 경계가 있을 수 있다.  

신뢰 경계의 중복과 데이터의 노출 증가는 악의적인 클라우드 소비자(사람 또는 자동화된)에게 IT 자원을 공격하거나 비즈니스 데이터를 훔치거나 손상시킬 수 있는 기회를 더 많이 제공한다.  

##### 3.4.2. 운영 관리 제어의 축소
<br/>
클라우드 소비자는 대개 온 프레미스된 IT 자원보다는 낮은 관리 제어 수준을 할당받는다. 클라우드와 클라우드 소비자간의 통신에 필요한 외부적인 연결과 클라우드 제공자가 어떻게 클라우드를 운영하는가는 이에 따른 위험 요소를 동반한다.  

+ 신뢰도가 낮은 클라우드 제공자는 클라우드 소비자에게 발행된 SLA의 보증사항을 유지하지 않을 것이다. 이는 클라우드 서비스에 의존하는 클라우드 소비자 솔루션의 품질을 저해한다.  
+ 클라우드 소비자와 클라우드 제공자 사이의 지리적인 거리가 멀수록 네트워크 흡수가 많아져 지연시간의 변동과 잠재적인 대역폭 제약이 있을 수 있다.  

SLA, 기술 점검, 모니터링과 결부된 법적 계약은 관리적인 위험요인과 이슈들을 완화시킨다. 클라우드 관리 시스템은 클라우드 컴퓨팅의 '서비스로서'의 특성에 따라 SLA를 바탕으로 구축된다. 클라우드 소비자는 실제 제공되는 서비스 수준을 계속 모니터링 하고 클라우드 제공자가 만든 품질 보증서를 계속 주지해야 한다.  

##### 3.4.3. 클라우드 제공자 사이의 제한된 이식성
<br/>
클라우드 컴퓨팅 업계 내에 구축된 업계 표준이 부족하여 공용 클라우드는 다양한 규모로 특허를 받는다.  

이런 환경에서 종속적인 주문 제작 솔루션을 보유한 클라우드 소비자가 한 클라우드 제공자에서 다른 제공자로 이동하는 것은 어려운 일이다.  

##### 3.4.4. 다중 영역 규제와 법적 이슈
<br/>
제3자인 클라우드 제공자는 더욱 저렴하고 편리한 지리적 장소에 데이터 센터를 구축할 것이다. 클라우드 소비자는 공용 클라우드가 제공하는 IT 자원과 데이터의 물리적 위치는 인식하지 못할 것이다. 어떤 조직에서는 업계나 정부의 데이터 프라이버시 및 저장소 정책에 관한 규제와 관련된 심각한 법적 고려사항에 대해 문제를 제기할 수 있다. 예를 들어 영국에는 영국 국민에게 속한 개인적인 데이터는 영국 내에 보관되어야 한다는 법이 있다.  

또한 잠재적 법적 이슈는 데이터에 대한 접근성 및 공개와도 관련이 있다. 어떤 국가에는 특정 형태의 데이터는 특정 정부에 공개되어야 한다는 법이 있다. 한 예로, 미국에 위치한 클라우드를 사용하는 유럽의 클라우드 소비자들의 데이터는 미국 정부 기관에 의해 유럽 내 위치한 다른 국가들에 위치한 데이터보다 쉽게 접근될 수 있다(미국 애국 법에 의해). 대부분의 규제의 골조는 데이터가 외부의 클라우드 제공자에게 위치해 있더라도 궁극적으로 클라우드 소비자 조직에 보안, 무결성, 데이터 저장소에 대한 책임이 있다고 인식하는 데 있다.  

+ 클라우드 환경은 다수의 클라우드 소비자가 IT 자원을 공유하는 중복된 신뢰 경계와 관련된 여러 가지 보안적인 어려움을 가져올 수 있다.  
+ 클라우드 소비자의 운영 관리는 플랫폼을 넘어 클라우드 제공자가 행사하는 제어로 인해 클라우드 환경에 제약을 받는다.
+ 클라우드 기반 IT 자원의 이식성은 개별 클라우드 각 특성에 의해 제한된다.
+ 제3자인 클라우드 제공자가 제공하는 데이터와 IT 자원의 지리적 위치는 클라우드 소비자의 제어를 벗어나며 다양한 법적, 규제적 고려사항을 야기한다.
